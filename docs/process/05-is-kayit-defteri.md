# 05 — İş Kayıt Defteri

> Eksen: **süreç**. Bu dosya Linear'ın repo içindeki takip aynasıdır. Her geliştirme,
> refactor, bug fix, dokümantasyon ve infra işi için tek satır tutulur. **Not kolonu kısa**:
> kapanan işlerde 1-2 cümle (ne yapıldı + commit hash + açık follow-up) — ayrıntı git history,
> Linear yorumu ve commit mesajında. Aktif (`Todo`/`In Progress`/`Review`) satırlarda planın
> özeti kalabilir.

Durumlar `Backlog`, `Sonraki Faz`, `Todo`, `In Progress`, `Review`, `Done`, `Canceled` setinden seçilir
(Linear karşılıkları ve "ayrı state olmayan" durumlar → [`04-otomatik-is-akisi-protokolu.md`](04-otomatik-is-akisi-protokolu.md) "Durum sözlüğü";
Demirkol takımında `Review` için ayrı state yok → Linear'da `In Progress` + notta belirtilir; `Blocked` için ayrı state yok → bloke iş `Todo`/`Backlog`'ta kalır + `blockedBy` ilişkisi + notta belirtilir).
Linear MCP erişilemiyorsa `Linear` alanına `MCP bekliyor` yazılır ve ilk uygun Linear temasında eşlenir.
